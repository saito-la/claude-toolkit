# person-research — 面談相手の徹底調査と情報整理

面談相手など人物の包括的情報を収集し、**簡潔・重複なし・根拠必須**で整理する。
毎回この長い指示を打たなくて済むよう、**本人であることが確実なURLを2-3個渡すだけ**で以下が自動的に走る。

## 入力

`$ARGUMENTS` に「本人であることが確実に確認できるURL（2-3個）」を受け取る。任意で氏名を併記してよい。

- URLが渡されていなければ、**氏名＋確実URLを求めて停止する**（推測で調査を始めない）。
- URLは「本人性のアンカー」。ここから外れる情報は原則採用しない。
- 対象人物について既知の一次記録（社内名簿・過去の面談メモ等）が手元にあれば、調査開始前にそれを確認し、基礎情報（氏名漢字・所属・職位・メール等）はそちらを優先する。

## 大原則

以下を厳守する。

1. **類推・推測を示さない。** 客観的事実は必ず出典URLで裏取りする。裏取りできない客観的事実は書かない。
2. **同姓同名の混入を絶対に避ける。** 全事実を「本人の指紋」に照合し、一致しないものは統合せず除外する。
3. **固有名詞（漢字氏名・および読み／ふりがな）を推測しない。** 漢字も読み（ふりがな・ローマ字）も、**出典に literally 現れた表記だけ**を転記する。
   - 漢字：slug・ローマ字・かな・読みから生成しない（romaji→漢字は一意でない＝捏造になる）。かな表記の名はかなのまま書く。
   - **読み／ふりがな：漢字氏名からの“よくある読み”を当てない（漢字が読めても読みは推測）。** 読みは researchmap／KAKEN／J-GLOBAL／公式ページに明記された値だけを採用し、無ければ「読み不明」として漢字のみ記載する。
   - 読み・学位・所属・年が不確かな場合も、出典未確認のまま慣用的な読みで確定しない。漢字が自明でも、読みは一次ソースで確認するまで書かない。
4. **読み・学位・所属・年が不確かなら researchmap / KAKEN / 公式ページの登録値を準一次ソースとして最優先する**（生成AIによる二次要約より優先）。確定できない読み・年は書かない。
5. **推測が許されるのは2項目だけ**：`年齢`（生年月日不明時。根拠を添えて「推定」と明記）と `興味の推移`（研究テーマ・活動の変化からの解釈。「推測」と明記）。それ以外は事実のみ。

## 手順

### Step 1. 指紋の確定

先に `WebFetch` を使う（deferred のため `ToolSearch: select:WebFetch,WebSearch` で取得してから）。渡された各URLを取得し、次を「本人の指紋（identity fingerprint）」として固定する：

- 氏名（出典どおりの表記。読みが明記されていれば読みも）
- 現所属・職位、専門分野
- 学位・出身、活動年代の手がかり（学部卒年・生年など）
- **一意ID**（同名排除の主軸）：researchmap ID／ORCID／KAKEN 研究者番号／J-GLOBAL ID／PubMed・Scopus author ID／所属機関の教員ページURL

IDが1つでも取れれば、以降の照合はそのIDを軸にする。

### Step 2. fan-out 調査

`Agent` ツール（`Explore` / `general-purpose`）を**並列起動**し、観点ごとに分担する。各サブエージェントには **Step 1 の指紋を必ず渡し**、「指紋に一致する人物の情報だけを、出典URL付きで収集せよ。一致確認できないものは『別人可能性あり』として報告せよ」と指示する。

1. **和文レジストリ**：researchmap／KAKEN(KAKENHI DB)／J-GLOBAL／CiNii／所属大学の教員データベース・researcher DB
2. **英文学術**：ORCID／PubMed／Scopus・Google Scholar／各論文DB（**1st author と last author は網羅的に**、それ以外は代表的なものを選定）
3. **経歴・委員・受賞**：学会役員・官公庁/学会委員歴・受賞歴・プレスリリース・大学広報・学位論文
4. **一般・報道・その他**：ニュース・インタビュー・講演・著書・（公開されていれば）家族構成・趣味関心

和文・英文の両方から集め、偏りを無くす。

### Step 3. 同姓同名ガード ＋ 反証検証

- 収集した各事実を指紋アンカー（所属系譜／分野／一意ID）に照合。一致しないものは**統合せず**、除外または「別人可能性」として明示する。
- 各客観的事実について「これは別の同名者の情報ではないか？」を**敵対的に**再確認する。出典URLの無い客観的事実は不採用。
- 漢字表記は出典に現れたものだけ。読み・年・学位が不確かなら researchmap/KAKEN/公式値を正とし、確定できないものは書かない。

### Step 4. レポート出力

下記スキーマで出力する。**冗長表現を避け、客観的事実の列挙に留める。** 文字数は情報量に応じて増減してよい。

```markdown
# 人物調査レポート：<氏名>

## 基本情報
- **氏名**：<出典どおりの表記／読み（判明時）>
- **年齢**：<実年齢。生年月日不明なら「推定：約XX歳（根拠：学部卒YYYY年から逆算）」と明記>

## 学歴
- <年：機関・学位。出典>

## 職歴
- <年：所属・職位。出典>

## その他の経歴（委員歴・受賞歴など）
- <項目。出典>

## 主要論文
### 筆頭・責任著者（1st / last author：網羅）
- <著者位置｜タイトル｜誌名 年｜DOI/URL>
### その他の代表的論文
- <同上（代表のみ）>

## 主要研究費獲得実績（研究代表者のみ）
- <課題名｜制度・種目｜期間｜（判明時）金額｜KAKEN等URL>

## 興味の推移（※研究テーマ・活動の変遷からの解釈・推測）
- <時期区分ごとの関心の移り変わり>

## 主要業績（特筆すべき成果・貢献）
- <項目。出典>

## 家族構成（公開情報がある場合のみ）
- <項目。出典>

## 趣味・関心事（公開情報がある場合のみ）
- <項目。出典>

## 出典一覧
- [1] URL
- [2] URL
...

## 確度・注記
- 同姓同名の切り分け根拠：<指紋のどの要素・IDで本人性を担保したか>
- 未確定・要確認事項：<読み不明・年不明など、推測で埋めなかった箇所>
```

### Step 5. 検証

以下を確認し、レポートを修正する：

- [ ] 同じ客観的事実が重複記載されていない（あれば削除）
- [ ] 各客観的事実に出典があり、推測に基づいていない（推測は `年齢`・`興味の推移` の2項目のみ、かつ明記済み）
- [ ] 漢字の固有名詞がすべて出典に存在する（slug/ローマ字/かなから生成した漢字が無い）
- [ ] 和文・英文の両方を参照し、偏りが無い
- [ ] 同姓同名の切り分け根拠が明記されている

調査結果を自分用の人物ノート（wiki・CRM等）に取り込む場合は、機密度に応じたラベル付けをしたうえで、確定した高確度の客観的事実のみを追記すること。

## 使い方

```
/person-research https://researchmap.jp/xxxxx https://www.example-u.ac.jp/faculty/yyy
/person-research 山田太郎 https://orcid.org/0000-0000-0000-0000 https://researchmap.jp/xxxxx
```

チャットで「この人を面談前に調べて」＋確実URL、でも同様に発火する。
