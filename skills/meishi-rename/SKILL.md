# meishi-rename

「名刺リネーム」「名刺のファイル名を直して」等の依頼、または ScanSnap 等でスキャンされた名刺PDFのファイル名が OCR 誤認識で読めない状態（例: `Mhlw_gAA.pdf`、`ILIPNIVERSi n_4,EPPt*J'Et4-'11,30fArgrc•.pdf`）のときに使うスキル。名刺画像を読み取り、`名刺 <氏名> <所属>.pdf` の形式にリネームする。

## 命名規則

- 形式：`名刺`＋半角スペース＋`<氏名>`＋半角スペース＋`<所属>`＋`.pdf`
- 氏名：名刺に印字された氏名をそのまま転記する（漢字表記があれば漢字、なければローマ字／カタカナ）。読み不明・判読不能な部分は推測しない。
- 所属：正式名称ではなく短い通称にする（対象フォルダの既存命名例に揃える）
  - 例：厚生労働省、○○大学、○○病院、○○製薬、AMED、PMDA
  - 「株式会社」「独立行政法人」「国立研究開発法人」等の法人格表記、部署名・役職名は原則省略する
  - 判断に迷ったら対象フォルダ内の既存「名刺 ～.pdf」ファイルを `ls` で確認し、命名スタイルを揃える
- 同一人物で所属の異なる名刺が複数ある場合は、所属名で自然に区別する
  - 例：`名刺 山田太郎 ○○大学.pdf` と `名刺 山田太郎 厚生労働省.pdf`

## 実行手順

1. 対象パスを確認する（未指定ならユーザーにスキャン保存先フォルダを確認する）。ファイル名が意味不明な英数字列（OCR誤認識）のPDFのみを対象にする。既に「名刺 」で始まるファイルは対象外。
2. 各PDFを Read ツールで読み取る（多ページの場合は表面・裏面の両方を確認し、氏名・所属機関を特定する）。
3. 同一の名刺が重複スキャン（複数ファイルに分割）されていないか確認する（氏名・所属が一致し、ファイルサイズが近い場合は要注意）。重複が疑われる場合は削除前に必ずユーザーに確認する（無断で `rm` しない）。ユーザーが削除を承認した場合はファイルサイズが大きい方（画質が良い目安）を残す。
4. 命名規則に従って新ファイル名を決め、`mv` でリネームする（同名衝突がある場合はユーザーに確認する）。
5. `ls` でリネーム結果を確認し、変更一覧をユーザーに報告する。
6. ユーザーが自分でファイル名をさらに修正した場合は、その差分を今後の命名判断に反映する。恒常的なパターン（略称の選び方など）と分かった場合は、このスキルの「命名規則」セクションに追記して定着させる。

## 注意

- 氏名・所属は名刺に印字された文字をそのまま転記する。判読不能な場合は推測せず、ユーザーに確認する。
- 名刺以外のPDF（契約書・証明書等）が混在するフォルダでは、明らかに名刺と分かるファイルのみを対象にする。
